iPhone Feedback Tool Hilfe

With the iPhone Feedback Tool the developers of iOS applications can be informed in a fast and comfortable way about errors or general requests from their users.

1. Annotations

When you shake the device, a screenshot of the current view is taken and presented in the annotations view. There you can mark certain points of interest in the screenshot and add notes. If there is personal information in the screenshot that is not supposed to be sent to the developers, you have the ability to black them out.

Back to top

1.1 Annotate

Using a long press or double tap generates a new annotation and presents a dislog for the notes text entry. The input dialog closes after writing the note and tipping on Done. When you tap on the annotations you are able to see the notes and amplify them. To move an annotation you can simply drag & drop them.

PLEASE NOTE: The numbering of the annotations is consecutive and is updated, when annotations in between get deleted.

PLEASE NOTE: There is a limit of max 9 annotations per screenshot.

Back to top

1.2 Black Out

With a tap on Edit the control elements for blacking out are displayed. There are 3 brush sizes and an eraser available. With a tap on Done the control elements are hidden and you get back to the annotations view.

Back to top

2. Message

This view is for general messages and notes that are too big for an annotation. Here you can also leave your change requests, compliments or any other kind of feedback.

Back to top

3. Options

In the options view you can define which information should be sent. Selectable items are contact details, screenshot, message and the stacktrace/log.

Back to top

3.1 Contact Details

These are the contact details that have been entered when the tool was used for the first time oder that were used the last time a feedback was sent. With a tap on Change a new set of contact details can be entered and saved for this and future feedback submissions.

Back to top

3.2 Screenshot

Just in case there is only a general message and no further information displayed on the taken screenshot, you can deselect it from the feedback data that will be submitted.

Back to top

3.3 Message

If there is no general message or when you do not want to send the entered one anymore, just deselct it..

Back to top

3.4 Stacktrace and Log

The iPhone Feedback Tool is able to retrieve and attach the stacktrace and application log to the feedback data.. This is very helpful to the developers when they ahve to deal with reported errors. With a tap on View this data can be checked, before it will be submitted to the developers.

Back to top

4. Submission

This view collects all the data, prepares them for submission and finally submits them. Depending on the submission mode that the developers have chosen, it might be necessary (like in native mail app), that all data is presented to the user and that he actively has to send them. The tool will close automatically after an information message containing the submission status has been displayed.
